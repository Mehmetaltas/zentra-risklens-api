# Security Policy

ZENTRA is a behavioral financial infrastructure project designed
with security and responsible disclosure principles.

---

## Supported Versions

The public repository represents a demonstration layer of the ZENTRA
platform. Sensitive infrastructure components and operational systems
are not publicly exposed.

Security issues should be reported responsibly.

---

## Reporting a Vulnerability

If you discover a potential security vulnerability related to the
ZENTRA platform or this repository, please report it privately.

Do NOT open a public issue.

Please send details to:

contact@zentrarisk.com

Include:

• description of the vulnerability  
• steps to reproduce  
• potential impact  

We will investigate and respond as quickly as possible.

---

## Security Principles

ZENTRA is designed with the following security principles:

• behavioral anomaly detection  
• fraud signal monitoring  
• account misuse observation  
• financial pattern analysis  
• explainable intelligence outputs  

The system aims to assist institutions in identifying financial
risk signals and abnormal patterns.

---

## Responsible Disclosure

We encourage responsible disclosure of vulnerabilities.

Security researchers who report valid issues responsibly will be
acknowledged for helping improve the platform.
